# Troubleshooting

Common issues and solutions for VPS VPN.

## Connection Issues

### Cannot Connect to VPN

**Symptoms:**
- Connection timeout
- "Handshake did not complete"

**Solutions:**

1. **Check if server is running:**
   ```bash
   docker-compose ps
   docker-compose logs
   ```

2. **Verify port is open:**
   ```bash
   # Check if port is listening
   netstat -ulnp | grep 51820
   
   # Test from client
   nc -vzu your-vps-ip 51820
   ```

3. **Check firewall:**
   ```bash
   # UFW
   ufw status
   ufw allow 51820/udp
   
   # iptables
   iptables -L -n | grep 51820
   ```

4. **Verify WG_HOST:**
   ```bash
   # Must be accessible from internet
   echo $WG_HOST
   ping $WG_HOST
   ```

### Web UI Not Loading

**Symptoms:**
- "Connection refused"
- Timeout on port 51821

**Solutions:**

1. **Check Web UI port:**
   ```bash
   docker-compose ps
   netstat -tlnp | grep 51821
   ```

2. **Check container logs:**
   ```bash
   docker-compose logs wg-easy
   ```

3. **Restart container:**
   ```bash
   docker-compose restart
   ```

## Performance Issues

### Slow VPN Speed

**Causes:**
- Server resource limits
- Network latency
- MTU issues

**Solutions:**

1. **Check server resources:**
   ```bash
   htop
   docker stats
   ```

2. **Adjust MTU:**
   ```bash
   # Lower MTU in .env
   WG_MTU=1400
   docker-compose down && docker-compose up -d
   ```

3. **Check network:**
   ```bash
   # Ping from client
   ping your-vps-ip
   
   # Trace route
   traceroute your-vps-ip
   ```

### High Latency

**Solutions:**

1. Choose server closer to your location
2. Use DNS servers closer to VPN server
3. Check for ISP throttling

## DNS Issues

### DNS Not Resolving

**Symptoms:**
- Cannot access websites
- "Server not found"

**Solutions:**

1. **Verify DNS in config:**
   ```ini
   [Interface]
   DNS = 1.1.1.1
   ```

2. **Try different DNS:**
   ```bash
   # In .env
   WG_DNS=8.8.8.8,8.8.4.4
   ```

3. **Flush DNS cache:**
   ```bash
   # Linux
   systemd-resolve --flush-caches
   
   # macOS
   sudo dscacheutil -flushcache
   
   # Windows
   ipconfig /flushdns
   ```

### DNS Leaks

**Check for leaks:**
- Visit https://dnsleaktest.com
- Compare DNS servers shown

**Prevent leaks:**
```ini
[Peer]
# Use VPN for all traffic
AllowedIPs = 0.0.0.0/0
```

## Client Issues

### Windows Client Problems

**"Unable to find %WINDIR%"**

1. Run as Administrator
2. Reinstall WireGuard

**TAP adapter errors**

1. Uninstall and reinstall WireGuard
2. Disable other VPN software

### Mobile Client Issues

**iOS: VPN permission denied**

1. Settings > VPN > Allow WireGuard
2. Reset network settings if persistent

**Android: No internet when VPN on**

1. Check "Exclude private IPs" option
2. Verify AllowedIPs in config

### Linux Client Issues

**"RTNETLINK: Operation not permitted"**

Run with sudo:
```bash
sudo wg-quick up wg0
```

**Module not loaded**

```bash
# Load wireguard module
sudo modprobe wireguard

# Make persistent
echo wireguard | sudo tee /etc/modules-load.d/wireguard.conf
```

## Server Issues

### Container Won't Start

**Error: "port already allocated"**

```bash
# Check what's using the port
netstat -ulnp | grep 51820

# Stop conflicting service or change port
```

**Error: "network not found"**

```bash
# Recreate network
docker network create vpn_network
docker-compose down && docker-compose up -d
```

### Lost Server Access

**If WireGuard blocks SSH:**

1. Use server console/KVM
2. Edit config: `AllowedIPs = 0.0.0.0/0,!YOUR_IP/32`
3. Restart service

### Persistent Keepalive Issues

**Connection drops frequently**

```bash
# Increase keepalive in .env
WG_PERSISTENT_KEEPALIVE=30

# Or disable if not behind NAT
WG_PERSISTENT_KEEPALIVE=0
```

## Configuration Errors

### Invalid Private Key

**Error: "invalid key"**

Private key must be:
- 44 characters base64
- Generated by `wg genkey`

```bash
# Generate valid key
wg genkey | tee privatekey
```

### Invalid Endpoint

**Error: "endpoint failed"**

Endpoint must be:
- Public IP or domain name
- Correct port

```bash
# Verify WG_HOST
echo $WG_HOST

# Test connectivity
nc -vzu $WG_HOST 51820
```

## Docker Issues

### Out of Memory

```bash
# Increase container memory
# Edit docker-compose.yml
deploy:
  resources:
    limits:
      memory: 256M
```

### Volume Permissions

```bash
# Fix permissions
sudo chown -R 1000:1000 config/
```

## Reset

### Complete Reset

```bash
# Stop container
docker-compose down

# Backup config
cp -r config config.backup

# Remove config
rm -rf config/*

# Start fresh
docker-compose up -d
```

### Reset Client

```bash
# Remove client via Web UI
# Or via CLI
./wg.sh remove-client client-name
```

| `ETIMEDOUT` | Firewall blocking |
